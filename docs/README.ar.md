## علم عدّ الآي وصلته بالقراءات

تُقرأ آيات القرآن الكريم بعشر قراءات متواترة، لكلّ قراءة راويان، فيكون مجموع الروايات عشرين رواية. فقراءة الإمام عاصم — مثلاً — رواها عنه حفص وشعبة.

## إشكالية اختلاف أرقام الآيات

لاختلاف أنظمة العدّ تختلف أرقام الآيات بين القراءات، فما يُعدّ آية مستقلة في عدّ قد يكون جزءًا من آية في عدّ آخر.

**مثال:** يعدّ العدّ الكوفي (حفص) الحروفَ المقطّعة ﴿الم﴾ في أوّل سورة البقرة آيةً مستقلة (الآية 1)، بينما يدمجها العدّ المدني الأخير (ورش وقالون) مع الآية التي تليها. فتصبح الآية الخامسة عند حفص هي الآية الرابعة عند ورش.

لذلك فإنّ مطابقة الآيات بين المصاحف بالاعتماد على رقم السورة ورقم الآية فقط تُعطي نتائج **خاطئة** في المواضع التي يختلف فيها العدّ.

## أبرز مواضع الاختلاف

تتركّز الفروقات بين أنظمة العدّ في ثلاثة مواضع:

**أوّلًا: البسملة في سورة الفاتحة**
يعدّها العدّ الكوفي والمكي آيةً مستقلة (الآية الأولى)، أمّا سائر الأعداد فلا تعدّها.

**ثانيًا: الحروف المقطّعة (فواتح السور)**
الحروف التي افتُتحت بها تسعٌ وعشرون سورة — نحو: ﴿الم﴾ و﴿الر﴾ و﴿حم﴾ — يعدّها العدّ الكوفي آيات مستقلة، بينما تدمجها بقية الأعداد — أو بعضها — مع ما يليها.

**ثالثًا: التعويض بالتقسيم في الفاتحة**
حين تُسقط الأعداد غير الكوفية البسملةَ من العدّ، تعوّض ذلك بتقسيم آية أخرى إلى آيتين، فيبقى مجموع آيات الفاتحة سبعًا بإجماع.

## ما يوفّره هذا المشروع

يوفّر هذا المشروع **ملفات JSON جاهزة للاستخدام** تربط أرقام الآيات بين أنظمة العدّ المختلفة، دون الحاجة إلى كتابة أي شيفرة برمجية.

## بنية البيانات

### ملفات الربط (`data/mappings/by-counting-system/kufi-to-{system}.json`)

يوفّر كلّ ملف ربطًا كاملاً آيةً بآية من حفص إلى نظام العدّ المستهدف، مُنظّمًا بحسب السور ثمّ بحسب رقم الآية في حفص:

<div dir="ltr">

```json
{
  "_version": "0.1.0",
  "_source": "kufi",
  "_target": "madani-last",
  "surahs": {
    "1": {
      "hafs_ayah_count": 7,
      "target_ayah_count": 7,
      "ayahs": { ... }
    }
  }
}
```

</div>

تحتوي كلّ سورة على `hafs_ayah_count` (عدد آيات حفص)، و`target_ayah_count` (عدد آيات النظام المستهدف)، وكائن `ayahs` مفهرس برقم آية حفص.

### أنواع ربط الآيات

تُربط كلّ آية من حفص بالنظام المستهدف بإحدى ثلاث طرق:

**1. مربوطة (mapped)** — تطابق مباشر (آية بآية). معظم الآيات من هذا النوع.

<div dir="ltr">

```json
{ "target_ayah": 4, "status": "mapped" }
```

</div>

**2. مدمجة (merged)** — لا يوجد لهذه الآية رقم مستقلّ في النظام المستهدف؛ محتواها مدمج مع آية مجاورة.

<div dir="ltr">

```json
{ "target_ayah": null, "status": "merged" }
```

</div>

مثال: يعدّ حفص ﴿الم﴾ الآية 1 في سورة البقرة. في العدّ المدني الأخير تُدمج ﴿الم﴾ مع ما يليها، فلا يكون لها رقم مستقلّ.

**3. مقسّمة (split)** — آية واحدة عند حفص تصبح آيتين (أو أكثر) في النظام المستهدف.

<div dir="ltr">

```json
{ "target_ayah": 5, "status": "split", "splits_into": [5, 6] }
```

</div>

`target_ayah` هو رقم الآية الأولى الناتجة. `splits_into` يسرد جميع أرقام الآيات المستهدفة.

### الخرائط العكسية (`data/mappings/by-counting-system/{system}-to-kufi.json`)

الخرائط العكسية تنطلق من النظام المستهدف عائدةً إلى الكوفي (حفص). كلّ مدخل مفهرس **برقم آية النظام المستهدف** ويبيّن ما يقابله من آيات حفص:

**1. مربوطة (mapped)** — آية واحدة تقابل آية واحدة عند حفص.

<div dir="ltr">

```json
{ "hafs_ayah": 4, "status": "mapped" }
```

</div>

**2. تغطّي عدّة آيات (covers_multiple)** — آية واحدة في النظام المستهدف تشمل محتوى آيتين أو أكثر متتاليتين عند حفص.

<div dir="ltr">

```json
{ "hafs_ayah": 1, "hafs_ayahs": [1, 2], "status": "covers_multiple" }
```

</div>

مثال: الآية الأولى من الفاتحة في العدّ المدني الأخير تشمل البسملة (آية حفص 1) و«الحمد لله رب العالمين» (آية حفص 2).

**3. ناتجة عن تقسيم** — عدّة آيات في النظام المستهدف تشير جميعها إلى آية حفص واحدة.

<div dir="ltr">

```json
{ "hafs_ayah": 7, "status": "mapped" }  // آية مستهدفة 6
{ "hafs_ayah": 7, "status": "mapped" }  // آية مستهدفة 7
```

</div>

### خرائط الرواة (`data/mappings/by-rawi/`)

البيانات نفسها لكن مفهرسة باسم الراوي. لكلّ راوٍ ملفان:

- `warsh-to-hafs.json` — إيجاد مقابل حفص لأي آية عند ورش
- `hafs-to-warsh.json` — إيجاد مقابل ورش لأي آية عند حفص

### بيانات الرواة (`data/rawis/{rawi}.json`)

كلّ ملف راوٍ هو مؤشّر خفيف يدلّ على ملف الربط الصحيح:

<div dir="ltr">

```json
{
  "_rawi": "warsh",
  "_qiraa": "nafi",
  "_counting_system": "madani-last",
  "_mushaf_id": 4,
  "_mapping_file": "mappings/by-counting-system/kufi-to-madani-last.json"
}
```

</div>

### عدد آيات السور (`data/surah-counts/{system}.json`)

عدد الآيات لكلّ سورة في كلّ نظام عدّ:

<div dir="ltr">

```json
{
  "_counting_system": "madani-last",
  "_total_ayahs": 6213,
  "surahs": { "1": 7, "2": 285, ... }
}
```

</div>

### الفروقات (`data/differences.json`)

البيانات العلمية الخام: فروقات فواصل الآيات على مستوى الكلمات بين كلّ نظام عدّ والمرجع الكوفي:

<div dir="ltr">

```json
{ "surah": 2, "hafs_ayah": 1, "word": "الم", "type": "merge" }
```

</div>

- `merge`: يعدّ الكوفي هذه الكلمة فاصلة آية، والنظام المستهدف لا يعدّها.
- `split`: يعدّ النظام المستهدف هذه الكلمة فاصلة آية، والكوفي لا يعدّها.
